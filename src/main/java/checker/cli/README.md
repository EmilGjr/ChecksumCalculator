# 💻 CLI Module - Интерфейс командной строки

## Описание
Этот модуль обрабатывает взаимодействие пользователя с программой через командную строку. Включает парсинг аргументов, обработку команд и вывод справки.

## Классы

### CommandLineOptions.java
**Парсер и хранилище опций** командной строки.
- Парсит аргументы в формате `-key=value`
- Валидирует режимы работы
- Предоставляет методы-предикаты для проверки режима
- Хранит все параметры программы

**Поддерживаемые аргументы:**
```
-m=<mode>          Режим: calculate, verify
-p=<path>          Путь до файла/папки для сканирования
-a=<algorithm>     Алгоритм: md5, sha256 (по умолчанию sha256)
-c=<file>          Путь до файла контрольных сумм
-t=<file>          Путь для сохранения результатов проверки
-nfl               Не следовать символическим ссылкам
--help             Показать справку
```

### ChecksumCalculatorMain.java
**Главная входная точка** приложения.
- Запрашивает у пользователя путь, если не указан
- Выводит приветствие и информацию о сканировании
- Создает фоновый поток для интерактивных команд
- Запускает основной процесс сканирования
- Сохраняет результаты в папку `checksums/`

**Интерактивные команды во время сканирования:**
```
pause      Приостановить сканирование
resume     Возобновить сканирование
quit       Завершить программу
```

### CommandLineInterface.java
**Демонстрационный класс** для вывода структуры файлов в консоль.
- Показывает иерархию директорий
- Используется для проверки построения дерева файловой системы

## Примеры использования

```bash
# С указанием пути (аргумент)
java -cp target/classes checker.cli.ChecksumCalculatorMain -p=/path/to/folder

# Без аргументов (запросит путь)
java -cp target/classes checker.cli.ChecksumCalculatorMain

# Проверка контрольных сумм
java -cp target/classes checker.cli.ChecksumCalculatorMain -m=verify -p=. -c=checksums/folder_sha256.txt

# Использование MD5
java -cp target/classes checker.cli.ChecksumCalculatorMain -p=/path -a=md5
```

## Поток выполнения

1. **Парсинг аргументов** → `CommandLineOptions.parse(args)`
2. **Запрос пути у пользователя** (если не указан)
3. **Вывод информации** о начале сканирования
4. **Запуск фонового слушателя** для команд паузы/возобновления
5. **Выполнение** основного процесса
6. **Ожидание** завершения слушателя команд
7. **Сохранение результатов** в `checksums/`

## Паттерны проектирования
- **Command**: Обработка различных команд (pause, resume, quit)
